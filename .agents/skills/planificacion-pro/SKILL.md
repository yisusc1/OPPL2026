# Planificación Pro
## Cuándo usar esta habilidad
- Cuando el usuario pida un plan paso a paso, una estrategia o una hoja de ruta.
- Cuando haya que entregar algo (landing, vídeo, proyecto, lanzamiento) con tiempos.
- Cuando el usuario tenga muchas tareas sueltas y quiera ordenarlas.
## Inputs necesarios (si faltan, pregunta primero)
1) Resultado final (qué significa “terminado”).
2) Fecha límite o ritmo (hoy, esta semana, sin prisa).
3) Recursos disponibles (herramientas, equipo, presupuesto, tiempo diario).
4) Criterios de éxito (qué debe cumplir para estar bien).
## Workflow
1) Define el resultado final en 1 frase y lista 3 criterios de éxito.
2) Divide el trabajo en fases (máximo 4):
- Preparación
- Producción / Ejecución
- Revisión / QA
- Publicación / Entrega
3) Dentro de cada fase crea:
- Tareas en orden
- Entregable claro (qué sale de esa fase)
- Tiempo estimado por tarea (aproximado)
4) Añade “Riesgos y mitigación” (3–5):
- Si pasa X → hago Y
5) Cierra con una checklist final de validación.
## Reglas de calidad
- Evita planes infinitos: prioriza lo que desbloquea lo siguiente.
- Si hay dependencias, indícalas (“esto depende de X”).
- Si el usuario es principiante, reduce pasos y da opciones simples.
- Si el usuario es avanzado, incluye optimizaciones y atajos.
## Output (formato exacto)
Devuelve siempre:
1) Resultado final + criterios de éxito
2) Plan por fases (con tareas, tiempos y entregables)
3) Riesgos y mitigación
4) Checklist final (lista de verificación)
